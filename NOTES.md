# Notes

Akasha Attractor was created to formalize terminal convergence as a first-class concept.

Key distinction:

- `structural_sink` = likely missing propagation
- `intentional_terminal` = valid endpoint by design

Initial purpose:
- preserve the meaning of attractors
- prevent Forge from overbuilding every terminal node
- study what it means for a system to tend toward an endpoint

Questions:
- How does an attractor differ from equilibrium?
- Can attractors be stable, unstable, cyclic, or strange?
- When should a terminal state influence future graph design?
